07 - Data Models New Field

Main Purpose

  • Learn how to create a new field in a data model
  • Learn about the different mappings/schemas/serializers that support data models

Guide to Completing

This is an important module because you learn about the inner workings of an Invenio data model. Prior to performing the module, I highly recommend getting a general understanding of how Invenio defines a "data model" and getting used to the terminology of serializer, loader, mappings, and schemas--which you may do here. This tutorial has brief, clear explanations for why you are doing each step.

If you are not familiar with Python's indentation rules, read this.

****If you are new to Python
Step 4: Python is not a "free form" language, which means that indentation of code blocks has to be specifically placed. If you are unsure what this means, you don't need to worry about it yet--just make sure that any code you put into a Python file(ends in .py) lines up with the code around it, and do not use the "tab" bar to do so. In the example below, the top version will work, the bottom version will give you errors because of the extra space before owner.

contributors = Nested(ContributorSchemaV1, many=True, required=True)
owner = fields.Integer()

contributors = Nested(ContributorSchemaV1, many=True, required=True)
 owner = fields.Integer()
