00 - Prerequisites

How I Completed 00

If you choose to use Vagrant, it will take a substantial amount of time to get the VM running and for everything to download.

Prior to this tutorial, I was not familiar with the git clone command. If you are also new to git clone, there are two ways to perform it (option 2 is much easier):

  1. Configure your Github account to use a new or existing SSH key
    1. If you have not generated an SSH key before, you must do that before 1.2. See here for how.
    2. If you have already generated an SSH key, or if you have already completed step 1.1, see here for how to connect it with your GitHub.

  2. Go to here and click on the "clone or download" button in the top right of the page
    1. Make sure what you clicked says "Clone with HTTPS".
    2. Copy and paste the HTTPS link to your terminal after typing "git clone", replacing the git clone command recommended in the tutorial.

Initially, I attempted to use Vagrant, which I had not used before. I ran into a host of issues:
  • My first attempt with my personal computer: after typing the git clone, my terminal froze up trying to download items.
  • My second attempt, using a mac, successfully allowed me to use vagrant up and vagrant ssh.
  • When I got to tutorial 01 - Getting Started, I was not able to run ./scripts/bootstrap. Which then in turn did not allow me to create ./scripts/setup or ./scripts/server. In order for these scripts to work I had to run additional scripts not mentioned in the tutorial.

I then switched over to VirtualBox and followed the prerequisite steps without issue!
