03 - Infrastructure Overview

Main Purpose

  • Learn the back-end, lower-level systems supporting Invenio Framework
  • Learn how to access information from back-end, lower-level systems

Guide to Completing 03

The commands you learn in this tutorial will not be used again in future tutorials, however if you would like to learn more about these lower-level systems check out these readings:
WSGI, Celery
Docker-compose.full.yml
RabbitMQ, Celery
Redis, Memcache

****If you are not sure where in the directory you should be
Step 1 is performed in ~/src/my-site
Step 4 is performed in ~/src/my-site

****If you are new to the command line/Bash shell
Steps 2, 3, and 7 will be using Bash shell commands that bring you to a new location within the directory. Your current environment is replaced with whichever environment is specified in the command. Step 2 is entering the database shell, step 3 is entering the cache shell, and step7 is entering the web-ui. After each of these steps, if you want to return to ~/src/my-site, you must use an exit command(or multiple exit commands). See below for exit commands, highlighted is what you type:

Step 2 to leave db shell: my-site-# \q
Step 2 to leave the root: root@‹combo of letters and numbers here›:/# exit

Step 3 to leave cache shell: 127.0.0.1:6379› exit
Step 3 to leave the root: root@‹combo of letters and numbers here›:/# exit

Step 7 to leave the root: root@‹combo of letters and numbers here›:/# exit
